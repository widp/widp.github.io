Viewing Logs And Log Viewers
January 4, 2023 · Tejaswi D Prakash

log viewing#

What I tried

  1. Logview mode emacs
  2. Notepad++ Notepad++ supports ‘User Defined Languages’ for key word highlighting, folding etc..
  3. Wrote a log search gui in tkinter

Tried this first , until I discovered notepad++ has all of the features I added built into it.

How I wanted to go about it

  • regular expression for the tokens and parsing for the log format turns out you don’t need that level of configuration See 1 timestamp tz LEVEL message etc..

Features I wanted

  1. Highlighting
    • Notepad++ ‘User defined language’ can be configured for keywords highlighting
  • Logview mode in emacs configuration

'(datetime-timezone 'Asia/Kolkata)
'(logview-additional-submodes
  '(("MYLOGS"
     (format . "NAME TIMESTAMP IGNORED LEVEL MESSAGE")
     (levels . "JUL")
     (timestamp)
     (aliases))))

  1. Filtering based on the different fields

    • Notepad++

    standard Notepad++ search

    • Emacs

    logview mode has keybindings and functions to filter by each of the above fields (Message , level etc..)

  1. folding Notepad++ folds can be defined for START and END keywords for a User definied language

    Still trying to figure out folding in emacs Although hideshow does a decent job, it doesn’t have the ‘fringe’ [+] [-] to expand and collapse

  2. tailing logs on Notepad++ ‘monitoring mode’ is built in, but this doesn’t seem to work for me (likely because logs are on network drive) Emacs has auto-revert-tail-mode but has the same issue
